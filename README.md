# Оболочка KinoPUB для Media Station X (LG webOS)

Современный фронтенд («оболочка») для **KinoPUB** в стиле Apple TV, работающий
внутри **Media Station X (MSX)** на телевизорах LG webOS (CX и подобные) — а
также на любой другой платформе, где запускается MSX (Tizen, Android TV,
Fire TV, браузеры).

Проект построен так, как MSX и предполагает расширять: размещённый на вашем
хостинге **плагин взаимодействия** (interaction plugin — «мозг»), который
обращается к официальному API KinoPUB и возвращает JSON для MSX, плюс
размещённый **видеоплагин** (плеер), который отвечает за воспроизведение,
докрутку с места остановки и синхронизацию прогресса. MSX сам нативно рисует
интерфейс из этого JSON — так мы наследуем его надёжную навигацию пультом,
систему фокуса и плеер, а оформляем всё в стиле tvOS: тёмная «стеклянная»
палитра, крупные скруглённые постеры, горизонтальные полки, кинематографичные
страницы с описанием и индикаторы прогресса.

> ⚠️ **Правовой статус / границы проекта.** Это только *клиентский интерфейс*
> к официальному API KinoPUB (документация на kinoapi.com). Он **не содержит
> контента** и **не содержит учётных данных**. Вам понадобится собственный
> аккаунт KinoPUB и собственная пара `client_id` / `client_secret` (запросить
> можно у `support@kino.pub`). Соблюдайте условия использования KinoPUB и
> законодательство вашей страны.

## Возможности → где реализованы

| Требование | Реализация |
|---|---|
| Авторизация устройства (OAuth2 Device Flow) | `js/api.js` (`requestDeviceCode`, `pollDeviceToken`, `refresh`) + `js/ui.js` `buildAuth` + автоопрос в `js/app.js` |
| Остаток дней подписки | `GET /v1/user` → показывается в поле **extension** (правый верхний угол) |
| Настройки устройства (поток, зеркало, 4K, HDR, качество, язык озвучки, субтитры) | `js/ui.js` `buildSettings`/`buildOptionChooser`, сохранение через `js/store.js` (TVXStorage) |
| Главная с полками (продолжить, новинки, популярное…) | `js/app.js` `buildHomeAsync` + `js/ui.js` `buildHome` |
| Полный список категорий (Фильмы, Сериалы, Мультфильмы, Аниме, Документальные, Концерты, 3D, ТВ-шоу) | `CONFIG.SECTIONS` + `handleSection` (типы + виртуальные жанровые разделы) |
| Карточки с постером, рейтингом, годом, качеством, прогрессом | `js/ui.js` `itemToCard` |
| Пагинация / «показать ещё» | `js/app.js` `paginate` (накопление страниц) |
| Закладки (избранное) | `bookmarkFolders`/`toggleBookmark` + `handleBookmarks` |
| История | `GET /v1/history` → маршрут `history` |
| Место остановки (сохранение + продолжение) | цикл `marktime` в плеере + `computeResume` (кнопка «Продолжить …») |
| Воспроизведение через плеер ТВ | **видеоплагин** `js/player.js`, запуск через `video:plugin:` |
| Прямой поток HLS/m3u8 | `GET /v1/items/media-links` → `chooseFile`/`streamUrl` |
| Выбор озвучки / субтитров / качества | качество и поток берутся из настроек; предпочитаемые **язык озвучки** и **субтитры** применяются к дорожкам `<video>`; смена «на лету» — сменить в настройках и продолжить |
| Релевантный поиск (название / актёр / режиссёр) | `GET /v1/items/search` + `?actor=`/`?director=`; нативная клавиатура MSX через `execute:code` с мостом на прокси |

## Структура репозитория

```
kinopub-msx/
├─ README.md
├─ .gitignore
├─ docs/
│  ├─ ARCHITECTURE.md      # потоки данных, таблица маршрутов, «рукопожатие» плеера
│  └─ INSTALL.md           # пошаговая установка (GitHub Pages + MSX + прокси)
├─ worker/
│  └─ worker.js            # Cloudflare Worker: CORS-прокси + клавиатура + прокси субтитров
└─ public/                 # всё, что вы хостите (статика)
   ├─ start.json           # Menu Root для MSX (левое меню в стиле tvOS)
   ├─ app.html             # хост плагина взаимодействия (невидимый)
   ├─ player.html          # хост видеоплагина (полноэкранный <video>)
   └─ js/
      ├─ config.js         # база/зеркала API, ключи, палитра, разделы, раскладка
      ├─ util.js           # ES5-хелперы (форматирование, SRT→VTT, query, debounce)
      ├─ store.js          # хранение токенов и настроек (TVXStorage/localStorage)
      ├─ api.js            # клиент API KinoPUB (авторизация, каталог, просмотр, закладки…)
      ├─ ui.js             # сборка JSON для MSX (внешний вид tvOS)
      ├─ app.js            # обработчик взаимодействия + маршрутизатор («мозг»)
      └─ player.js         # видеоплагин MSX (докрутка + marktime + дорожки)
```

Весь код `public/js/*.js` написан на **ES5 намеренно** — большинство браузеров
на ТВ не поддерживают ES6.

## Как это работает (в одном абзаце)

MSX загружает `start.json` как левое меню. Выбор пункта вызывает действие вида
`content:request:interaction:home@…/app.html`. MSX один раз загружает `app.html`
в скрытый iframe и вызывает `handleRequest("home", data, callback)` плагина.
Маршрутизатор в `app.js` обращается к API KinoPUB (`api.js`), превращает ответ в
JSON-контент MSX (`ui.js`) и возвращает его через `callback`. Нажатие на постер
вызывает `content:request:interaction:item:<id>@…` — строится кинематографичная
страница с описанием. Нажатие **Смотреть/Продолжить** вызывает
`video:plugin:…/player.html?mid=…&pos=…`; MSX загружает `player.html`, который
получает свежие ссылки на поток, перематывает на место остановки, играет и
периодически сохраняет прогресс через `marktime`. Полная таблица маршрутов — в
`docs/ARCHITECTURE.md`.

## Ключевые структуры JSON для MSX

**Пункт бокового меню** (`start.json`)
```json
{ "icon": "movie", "label": "Фильмы",
  "data": "request:interaction:section:movie@https://HOST/public/app.html" }
```

**Полка на главной** (строит `ui.buildHome`)
```json
{ "type": "list",
  "headline": "KinoPUB",
  "extension": "Иван · {ico:msx-white:schedule} 12 дн.",
  "background": "https://…/big.jpg",
  "pages": [
    { "headline": "Продолжить просмотр",
      "items": [ /* постеры-карточки 2×3 внутри сетки 12 колонок */ ] }
  ] }
```

**Карточка-постер** (`ui.itemToCard`)
```json
{ "type": "default", "round": true, "layout": "0,0,2,3",
  "image": "https://…/medium.jpg", "imageFiller": "cover",
  "title": "Оппенгеймер", "titleFooter": "2023",
  "stamp": "★ 8.1", "badge": "4K",
  "progress": 0.42, "progressColor": "#0A84FF",
  "action": "content:request:interaction:item:12345@https://HOST/public/app.html" }
```

**Действие воспроизведения** (кнопка на странице → видеоплагин)
```
video:plugin:https://HOST/public/player.html?mid=98765&id=12345&video=1&pos=738&type=movie
```

**Экран авторизации устройства** (`ui.buildAuth`) показывает `user_code` и
`verification_uri` (`kino.pub/device`) и сам опрашивает статус активации.

4. **Страница тайтла** — hero-страница, фильм против сериала, сезоны/серии, вычисление места остановки. *(готово)*
5. **Воспроизведение** — видеоплагин, выбор качества/потока, докрутка, `marktime`. *(готово)*
6. **Библиотека** — закладки, история, «продолжить смотреть», переключатели списка/просмотрено. *(готово)*
7. **Поиск** — мост нативной клавиатуры, поиск по названию/актёру/режиссёру, пагинация. *(готово)*
8. **Настройки** — поток/зеркало/качество/4K/HDR/озвучка/субтитры, с сохранением. *(готово)*

## Установка (кратко)

Полные шаги — в `docs/INSTALL.md`. Вкратце:

1. **Разместите `public/`** на любом статическом HTTPS-хостинге. Проще всего —
   **GitHub Pages** (бесплатно, HTTPS «из коробки»); подробная инструкция в
   `docs/INSTALL.md`. Запомните URL файла `app.html`.
2. **Разверните прокси** `worker/worker.js` (Cloudflare Worker) и укажите его URL
   в `public/js/config.js` → `API_BASE`. Прокси решает вопрос CORS, работу
   поисковой клавиатуры и загрузку субтитров.
3. Укажите свою пару **`CLIENT_ID`/`CLIENT_SECRET`** от KinoPUB. Для публичного
   хостинга (GitHub Pages) храните их **в секретах Worker**, а в `config.js`
   оставьте пустыми (см. INSTALL).
4. В `start.json` замените все `https://YOURHOST/kinopub-msx/public/app.html` на
   реальный URL вашего `app.html`.
5. На ТВ откройте **Media Station X → Settings → Start Parameter** и укажите ваше
   меню — одним из способов:
   - `menu:https://HOST/public/start.json`  *(статичное меню)*, **или**
   - `menu:request:interaction:menu@https://HOST/public/app.html`  *(динамическое
     меню, знающее о входе, которое строит плагин)*.
6. Перезапустите MSX. При первом запуске появится экран активации — введите код
   на `kino.pub/device`; приложение войдёт автоматически и откроет главную.

**Рекомендуемая настройка MSX:** включите *Rounded Style* — для скруглённых
постеров в стиле tvOS, которые используются повсюду.

## Честные ограничения

- **CORS:** плагин MSX работает в браузере ТВ и не может обращаться к
  `api.service-kp.com` напрямую, пока тот не отдаёт разрешающие CORS-заголовки;
  включённый прокси решает это. Сами видеопотоки играют кросс-доменно без
  прокси; через прокси идут только субтитры.
- **Смена дорожек «на лету»** зависит от платформы: предпочитаемые язык озвучки
  и субтитры применяются сразу, а нативный плеер webOS часто и сам показывает
  меню дорожек. Гарантированно рабочий путь — *Назад → сменить в Настройках →
  продолжить* (воспроизведение начнётся с сохранённого места).
- **`mid` (идентификатор медиа):** для воспроизведения нужен идентификатор
  конкретного видео/серии из `/v1/items/{id}`. Если в редком ответе его нет,
  тайтл сообщит «нет идентификатора медиа».
- Имена полей соответствуют документации KinoPUB v1.3; пара эндпоинтов
  (переключение закладки, notify устройства) в разных развёртываниях принимают
  чуть разные форматы — они вынесены в `api.js` для лёгкой правки.
